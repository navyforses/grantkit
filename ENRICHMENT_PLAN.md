## 1. არსებული მდგომარეობის აუდიტი

GrantKit-ის მონაცემთა ბაზაში ამჟამად **630 აქტიური გრანტია**. აუდიტმა გამოავლინა მნიშვნელოვანი ხარვეზები:

| ველი | ცარიელი ჩანაწერები | პროცენტი | შეფასება |
|------|-------------------|----------|----------|
| **ვებსაიტი** | 630 / 630 | 100% | კრიტიკული — მომხმარებელს არსად მიმართოს |
| **ტელეფონი** | 630 / 630 | 100% | კრიტიკული |
| **ელ-ფოსტა** | 630 / 630 | 100% | კრიტიკული |
| **თანხა** | 630 / 630 | 100% | კრიტიკული — მომხმარებელმა არ იცის რამდენს მიიღებს |
| **უფლებამოსილება (eligibility)** | 305 / 630 | 48% | ნაწილობრივი |
| **აღწერა < 30 სიმბოლო** | 278 / 630 | 44% | ცუდი — "Scholarship", "Lower Prices" ტიპის ჩანაწერები |

აღწერის სიგრძის განაწილება:

| სიგრძე | რაოდენობა | პროცენტი |
|--------|-----------|----------|
| < 20 სიმბოლო (ძალიან მოკლე) | 182 | 29% |
| 20–50 სიმბოლო (მოკლე) | 110 | 17% |
| 51–150 სიმბოლო (საშუალო) | 83 | 13% |
| 151–500 სიმბოლო (კარგი) | 171 | 27% |
| > 500 სიმბოლო (ვრცელი) | 84 | 13% |

**დასკვნა:** გრანტების **46%-ს** აქვს 50 სიმბოლოზე ნაკლები აღწერა, რაც მომხმარებლისთვის პრაქტიკულად უსარგებლოა.

## 2. გამდიდრების გეგმა — ორი ფაზა

### ფაზა A: LLM-ით ავტომატური გამდიდრება (AI Research)

**იდეა:** თითოეული გრანტისთვის LLM-ს ვთხოვთ ინტერნეტში მოიძიოს ორგანიზაციის ინფორმაცია და შეავსოს ცარიელი ველები.

**რას შეავსებს AI:**

| ახალი/შესავსები ველი | აღწერა | მაგალითი |
|----------------------|--------|---------|
| `website` | ორგანიზაციის ოფიციალური საიტი | `https://180medical.com` |
| `email` | საკონტაქტო ელ-ფოსტა | `info@180medical.com` |
| `phone` | საკონტაქტო ტელეფონი | `+1-877-688-2729` |
| `amount` | დაფინანსების თანხა/დიაპაზონი | `$500–$5,000` ან `Varies` |
| `description` (გაფართოება) | სრული აღწერა 150–300 სიტყვა | რას აკეთებს ორგანიზაცია, რა პროგრამა აქვს, ვის ეხმარება |
| `eligibility` (გაფართოება) | ვინ შეიძლება მიმართოს | ასაკი, დიაგნოზი, ლოკაცია, ფინანსური მდგომარეობა |
| `applicationProcess` (**ახალი**) | განაცხადის პროცესი | `Online application`, `Phone call`, `Referral required` |
| `deadline` (**ახალი**) | ვადა/სეზონურობა | `Rolling`, `March 31, 2026`, `Quarterly` |
| `fundingType` (**ახალი**) | დაფინანსების ტიპი | `One-time`, `Recurring`, `Reimbursement` |
| `targetDiagnosis` (**ახალი**) | სამიზნე დიაგნოზი/მდგომარეობა | `Cancer`, `Autism`, `General`, `Rare Disease` |
| `ageRange` (**ახალი**) | ასაკობრივი დიაპაზონი (სტრუქტურირებული) | `0-18`, `18-65`, `0-100` |
| `geographicScope` (**ახალი**) | გეოგრაფიული არეალი (დეტალური) | `Nationwide`, `Ohio only`, `International` |
| `documentsRequired` (**ახალი**) | საჭირო დოკუმენტები | `Medical records`, `Tax returns`, `ID` |
| `b2VisaEligible` (**ახალი**) | B-2 ვიზის მფლობელთათვის ხელმისაწვდომობა | `Yes`, `No`, `Uncertain` |

**პროცესი:**
1. 630 გრანტი იყოფა ბეჩებად (50 გრანტი ერთ ბეჩში)
2. თითოეულისთვის LLM-ს ეგზავნება ორგანიზაციის სახელი + არსებული აღწერა
3. LLM აბრუნებს სტრუქტურირებულ JSON-ს ყველა ველით
4. შედეგი ვალიდირდება და ჩაიწერება ბაზაში
5. ახალი აღწერები ითარგმნება 4 ენაზე (ka, fr, es, ru)

**სავარაუდო დრო:** 2–3 საათი (LLM გამოძახებები + ვალიდაცია + თარგმანი)

### ფაზა B: ახალი ფილტრები და სორტირება

| **განაცხადის მეთოდი** | Checkbox group | ონლაინ, ტელეფონით, რეფერალით, ფოსტით | საშუალო |

## 3. გრანტის დეტალური გვერდის გაუმჯობესება

ამჟამად გრანტის გვერდი აჩვენებს: სახელი, ორგანიზაცია, აღწერა, უფლებამოსილება, კატეგორია, ქვეყანა, ტიპი.

**შემოთავაზებული ახალი სექციები:**

1. **"მოკლე მიმოხილვა" ბარათი** — ორგანიზაციის შესახებ 2-3 წინადადება, რას აკეთებენ
2. **"დაფინანსების დეტალები"** — თანხა, ტიპი (ერთჯერადი/განმეორებადი), ვადა
3. **"ვინ შეიძლება მიმართოს"** — სტრუქტურირებული: ასაკი, დიაგნოზი, ლოკაცია, B-2 ვიზა
4. **"როგორ მივმართოთ"** — განაცხადის პროცესი, საჭირო დოკუმენტები, ბმული
5. **"საკონტაქტო ინფორმაცია"** — ვებსაიტი, ტელეფონი, ელ-ფოსტა (ამჟამად ცარიელია)
6. **"სტატუსის ინდიკატორი"** — ღიაა/დახურულია, ვადის ამოწურვამდე დარჩენილი დრო
